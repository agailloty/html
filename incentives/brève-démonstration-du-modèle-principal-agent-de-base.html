2 Brève démonstration du modèle principal agent de base

Le modèle de base considère qu’un principal désire déléguer à un agent la production de \(q\) unités de bien. La valeur que tire le principal de ces \(q\) unités de bien est \(S(q)\).
\(S(q)\) est une fonction croissante et concave, ce qui signifie que l’utilité marginale que le principal tire d’une unité supplémentaire de bien \(q\) est positive mais décroissante.
Le coût de production de l’agent est inobservable par l’agent mais il est de connaissance commune que ce coût fixe est \(F\) et le coût marginal \(\theta\) peut être élevé ou faible : \(\theta_1\) et \(\theta_0\). La probabilité que l’agent est efficient (\(\theta_1\)) est \(v\) et la probabilité qu’il soit inefficient (\(\theta_0\)) est \((1-v)\).

Le modèle de base assure que l’information est complète, il n’y a pas d’asymétrie d’information entre le principal et l’agent. La production efficiente dans cette configuration est obtenue en égalisant l’utilité marginale du principal à la production marginale de l’agent.

\[S'(q_1) = \theta_1 \] et \[S'(q_0) = \theta_0\]
La production obtenue lorsque l’agent est efficient est supérieure à la production obtenue lorsqu’il est inefficace.

\[W_1 > W_0\] avec \[W_1 = S(q^*_1) - \theta_1*q^* - F\] et \[W_2 = S(q^*_2) - \theta_2*q^* - F\]

Avant d’accepter le contrat, l’agent s’assure qu’il tirer un intérêt dans la tâche. Le principal lui fait donc un transfert tel que \(t\) tel que \[t_1 - \theta_1q_1 \geq 0\] et \[t_2 - \theta_0q_0 \geq 0\]

Dans ce modèle de base, le principal offre une liste de contrats avant de savoir à quel type d’agent il a affaires. Sachant qu’il y a deux types d’agent il calcule les bénéfices de chaque contrat qu’il va élaborer.

Son programme d’optimisation est donc le suivant :

\[max_{(t_1, q_1), (t_0, q_0) } v(S(q_1) - t) + (1-v)(S(q_0 - t_0))\]
